# UI Design Guidelines: CoolBox-Mobile

To maintain a premium, state-of-the-art visual experience in line with the Architect's vision, all UI changes must adhere to the following rules:

## 1. Color Palette & Aesthetics
- **No Lavender/Pale Purple**: Explicitly forbidden. Use clean whites, subtle greys, or curated dark mode tones.
- **Glassmorphism**: Use subtle transparency and blurring for overlays where appropriate.
- **Gradients**: Use smooth, professional gradients instead of flat colors for primary surfaces.

## 2. Interactive States
- **High-Contrast Selection**: The selected state must be distinct and immediately recognizable (high contrast).
- **Responsive Feedback**: Every interaction (click, hover, long press) must have a visual micro-animation or state change.

## 3. Layout & spacing
- **Uniform Distribution**: Ensure `Weight` (in Row/Column) and `Padding` are balanced and symmetrical.
- **Visual Hierarchy**: Use typography (Google Fonts like Outfit/Inter) and font weight to lead the user's eye to the most important information.

## 4. Components
- **Consistency**: All `Card` components must use a consistent corner radius (e.g., `12.dp`) and subtle borders.
- **No Placeholders**: Never ship placeholder images. Always use high-quality generated assets or clean iconography.
